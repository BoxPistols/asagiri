## Spacing Scale

Asagiri uses a consistent spacing scale based on rem units:

| Token | Value | Pixels (16px base) |
|-------|-------|-------------------|
| `--spacing-1` | 0.25rem | 4px |
| `--spacing-2` | 0.5rem | 8px |
| `--spacing-3` | 0.75rem | 12px |
| `--spacing-4` | 1rem | 16px |
| `--spacing-5` | 1.25rem | 20px |
| `--spacing-6` | 1.5rem | 24px |
| `--spacing-8` | 2rem | 32px |
| `--spacing-10` | 2.5rem | 40px |
| `--spacing-12` | 3rem | 48px |

## Border Radius

| Token | Value | Description |
|-------|-------|-------------|
| `--border-radius` | 0.375rem | Default border radius (6px) |
| `--border-radius-sm` | 0.25rem | Small radius (4px) |
| `--border-radius-lg` | 0.5rem | Large radius (8px) |
| `--border-radius-full` | 9999px | Fully rounded |

## Usage

### In CSS

```css
.my-component {
  color: var(--color-text);
  background: var(--color-surface);
  padding: var(--spacing-4);
  border-radius: var(--border-radius);
}
```

### Overriding Tokens

```css
:root {
  --color-primary: #your-color;
  --border-radius: 1rem;
}
```

### Dark Mode

```css
[data-theme="dark"] {
  --color-background: #0f0f0f;
  --color-text: #e4e4e7;
}
```

## Best Practices

1. **Always use tokens** instead of hardcoded values
2. **Don't create new tokens** without checking existing ones first
3. **Maintain consistency** across light and dark themes
4. **Test accessibility** when modifying color tokens
5. **Document custom tokens** with clear comments
